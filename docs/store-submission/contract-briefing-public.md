## 2. プラグイン概要

### 提供する決済機能
日本円ステーブルコイン **JPYC** (= 資金決済法第 2 条第 5 項に基づく**電子決済
手段**、 JPYC 株式会社発行 / 関東財務局長第 00099 号 資金移動業者) によるカート
決済機能を、 EC-CUBE 加盟店サイトに組み込むプラグインです。

### 法令上の重要事項
- JPYC は **電子決済手段** であり、 **暗号資産ではありません** (= 法令上明確に区別)
- 加盟店側は、 弊社 (uniple) が PSP として介在する設計のため、 **電子決済手段
  等取引業 (資金決済法第 2 条第 10 項) の登録は不要**となります
- JPYC Prepaid (= 旧 JPYC、 前払式支払手段) とは別ブランドであり、 本プラグイン
  は新 JPYC (= 電子決済手段) のみを対象としています

## 3. 決済フロー (= PCI DSS 非保持化の根拠)

### 加盟店 EC-CUBE サーバ ⇄ 顧客 ⇄ uniple ⇄ blockchain の責任境界

```
[顧客ブラウザ] ─→ [加盟店 EC-CUBE]                   [uniple Hosted Checkout]                [blockchain]
     │                  │                                       │                                  │
     │   1. 注文確定     │                                       │                                  │
     │ ─────────────→   │                                       │                                  │
     │                  │  2. createSession                     │                                  │
     │                  │ ─────────────────────────────────────→│                                  │
     │                  │  3. checkoutUrl 返却                  │                                  │
     │                  │ ←─────────────────────────────────────│                                  │
     │  4. uniple へ redirect (= 加盟店 server を経由しない)                                          │
     │ ──────────────────────────────────────────────────────→  │                                  │
     │  5. wallet 接続 + 署名 (= JPYC 残高確認、 加盟店 server を経由しない、                          │
     │      クレジットカード情報も発生しない)                       │                                  │
     │ ←──────────────────────────────────────────────────────  │                                  │
     │                                                          │   6. on-chain Tx broadcast       │
     │                                                          │ ────────────────────────────────→│
     │                                                          │   7. Tx confirm                  │
     │                                                          │ ←────────────────────────────────│
     │                                          8. webhook 配信 (= 加盟店 server に完走通知)         │
     │                  │ ←─────────────────────────────────────│                                  │
     │                  │  9. order status = 入金済み                                              │
     │  10. successUrl 着地 → サンクスページ                                                        │
     │ ←─────────────────────────────────────────────────────  │                                  │
```

### 加盟店 EC-CUBE サーバを通過するデータ
- 注文情報 (= 注文 ID、 決済合計額、 顧客情報)
- uniple checkout Session ID
- webhook payload (= 完走通知、 Tx hash 等の chain 上 evidence)

### 加盟店 EC-CUBE サーバを **通過しないデータ**
- **クレジットカード情報** (= 本プラグインはクレジットカード決済ではなく、 JPYC
  決済のため、 そもそもカード情報が流通しません)
- **wallet 秘密鍵 / signing secret** (= wallet で署名 → blockchain で verify、
  uniple サーバも知り得ません)

### PCI DSS 非保持化の整理
- そもそもクレジットカード情報を扱わないため、 PCI DSS の対象外となります
- ただし、 「決済情報の非保持化」 という EC-CUBE 公式ガイドラインの精神に整合
  するよう、 wallet 秘密鍵 / signing secret は加盟店 server を経由しないリダイ
  レクト型 + webhook 型の設計を採用しています

## 4. uniple 側の責任分担

| 項目 | 責任主体 |
|---|---|
| Hosted Checkout UI 提供 | uniple |
| wallet 接続 (= WalletConnect v2 / MetaMask / HashPort) | uniple |
| chain への Tx broadcast + confirm 監視 | uniple |
| webhook 配信 (= 完走通知、 約 3 日間の retry) | uniple |
| 加盟店向け credentials 発行 (= API key / Webhook secret、 admin UI で self-serve) | uniple |
| KYC (= 加盟店 onboarding) | uniple (= JPYC 株式会社の KYC を上流で活用) |
| AML / トラベルルール対応 | uniple |
| JPYC 残高 / 円転 | uniple (= JPYC EX 経由) |

## 5. 加盟店 (= 本プラグイン利用者) の責任分担

| 項目 | 責任主体 |
|---|---|
| EC-CUBE サーバ運用 (= HTTPS、 webhook 受信端点) | 加盟店 |
| プラグインインストール / 設定 (= API key 投入) | 加盟店 |
| 注文 / 商品管理 | 加盟店 |
| 顧客対応 (= 一次窓口、 必要に応じて uniple サポートにエスカレ) | 加盟店 |
| 自動返金不可のため、 加盟店から購入者へ JPYC 直送で返金対応 | 加盟店 |

## 6. 商標 / プレスキット遵守

- 「JPYC」 は JPYC 株式会社の登録商標であり、 プラグイン内で使用する際は
  プレスキット (= JPYC Logo Guideline v1.1, 2025-08-12 制定) に準拠しています
- プラグイン設定画面 + README に、 プレスキット準拠の **必須 3 行免責**を明示
  しています
- プラグイン独自アイコン (= JPYC Blue `#16449A` の derivative デザイン) を
  使用し、 JPYC ロゴ自体は使用していません (= プレスキット §6 「ロゴ変形・
  装飾追加 NG」 を厳格に遵守)

## 7. ご相談したい事項

### 7.1 商務条件
- 決済プラグイン公開に必要となる**基本契約および NDA** の条件 / 締結手続きに
  ついて、 ご教示いただきたく存じます
- 公式オーナーズストアにおける販売モデル (= 無料 / 有料) と、 有料の場合の手数
  料率
- 加盟店側への課金モデルとの併用可否

### 7.2 技術条件
- **PCI DSS 非保持化対応**: 上記 §3 の決済フロー図にて、 クレジットカード情報
  非経由および wallet 秘密鍵非経由の設計を採用済み
- **サンドボックス提供**: 審査担当者向けに、 テスト環境 URL / Test API キー /
  Test wallet 情報を別途、 機密情報として送付させていただく予定です
- **保守 SLA**: 弊社にてサポート期間 / 一次応答時間を策定し、 公式ストア掲載
  時にあわせて開示いたします

### 7.3 法令 / 規制
- **JPYC = 電子決済手段** の説明 (= 上記 §2 法令上の重要事項参照)
- **加盟店登録不要の根拠**: uniple PSP 介在モデル (= uniple が JPYC EX 経由
  で円転 / 即時決済代行)
- **AML / KYC**: 弊社にて完結 (= JPYC 株式会社の KYC を上流活用、 加盟店は別途
  登録不要)
